## 13. 版本发布信息与检查清单（Release & Metadata）

> 目标：当准备对外发布新版本（如 Demo、公测、正式版）时，有一份统一的检查清单，确保「玩家看到的所有信息」在游戏内外保持一致，不出现版本号/文案/截图错乱。

### 13.1 发版范围记录

- 每次计划发布新版本前，在本小节简单记录：
  - **版本号**：与工程配置与发布包 Tag 一致。
  - **发布日期**：预期/实际日期。
  - **本次目标**：例如「首个公开 Demo」「新增 Boss」「补完升级池」等。
  - **本次不包含的内容**：例如「暂不开放永久养成」「暂不做网络排行」等，避免玩家误解。

> 当需求调整时，应先更新本节的发版范围，再去改其他章节（关卡、敌人、升级、UI 等）的 GDD 内容。

#### v0.2.1

- **版本号**：`0.2.1`
- **发布日期**：2026-03-16
- **本次目标**：
  - 完成版本发布信息与检查清单（GDD 第 13 章），规范发版流程。
  - 对齐平台与发布章节，明确以本地导出为主。
- **本次不包含的内容**：
  - 仍不包含永久养成系统。
  - 仍不包含网络排行榜及在线功能。

#### v0.2.2

- **版本号**：`0.2.2`
- **发布日期**：2026-03-17
- **本次目标**：
  - 引入「2 条命」离散生命值系统，并在 HUD 中清晰展示剩余生命数。
  - 调整结算规则为「生命归零时立刻结算本局」，强化死亡反馈。
  - 新增「每波结束自动恢复 1 命（不超过 2 条）」机制，在保证惩罚感的前提下提高长战局容错率。
  - 对齐 README / About / GDD 文案，让对外简介与当前玩法一致。
- **本次不包含的内容**：
  - 未新增新的敌人、Boss 或升级词条，仅围绕生命系统与 UI 做迭代。

### 13.2 游戏内信息检查（About / 菜单）

- **About / Credits / 版本信息界面**

  - 游戏名称与 README / 商店标题一致（大小写与标点一致性）。
  - 版本号与本次构建版本一致（例如 `v1.0.0`）。
  - 一句话简介与对外简介方向一致（例如「竖屏单手 Roguelite 射击」）。
  - 制作人员名单与团队名称正确；第三方素材/插件按各自要求署名。
  - 版权年份范围更新，例如 `© 2024–2026 <开发者名>`。
- **主菜单 / 暂停菜单 / 设置**

  - 主菜单或明显位置展示当前版本号。
  - 若有「反馈 / Bug 反馈 / 社群 / 官网」按钮，链接正确可点。
  - 若有隐私或用户协议入口，说明与实际实现（是否联网、是否收集数据）一致。

### 13.3 仓库与文档检查（README / GDD / 已知问题）

- **README（根目录）**

  - 简介与 GDD 概览一致（平台、类型、核心卖点）。
  - 显示的版本号或 Release 徽章指向当前要发的版本。
  - 本地运行与打包步骤可实际执行成功（包括 Godot 版本与导出模板说明）。
  - 截图 / GIF：
    - 使用当前版本 UI（包含 Start Menu / Playing / Upgrade 等关键界面）。
    - 图片清晰、大小合适，放在 `docs/picture` 或约定的资产目录中。
- **GDD 与已知问题**

  - 各章节（关卡、敌人、升级、UI 等）描述与本次版本实现一致，不写超前大量「未来计划」当成已实现内容。
  - `KNOWN_ISSUES.md` 中列出本版本接受的已知问题，每条至少包括：
    - 简要描述；
    - 影响范围（是否影响新手体验、是否可能导致崩溃等）；
    - 是否为阻断发版的问题（Blocker / Non-blocker）。

### 13.4 发布渠道与对外页面检查（如有）

- **商店 / 平台页面（如 Itch / TapTap / 官网）**

  - 标题、副标题、分类标签与 GDD 的定位一致（例如「纵版 Roguelite 射击」「单手竖屏」等）。
  - 简短描述与 README / 游戏内 About 的核心信息一致。
  - 使用与 README 一致的一组截图，来源于稳定构建版本。
- **版本号与更新日志**

  - 商店中的版本号与构建版本 / 仓库 Tag 一致。
  - 更新日志仅描述本次版本真正完成的内容，不写尚未做完的功能。
- **法律与合规（如适用）**

  - 隐私政策与游戏内说明一致。
  - 若有在线功能，简单说明数据使用情况，并与真实实现保持一致。
- **导出文件相关配置**
  - Build 配置 也要同步，打包到各个平台的版本号要同步，比如安卓安装包要显示正确版本。
