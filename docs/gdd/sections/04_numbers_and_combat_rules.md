# 04 数值框架与战斗规则（MVP 简化版）

## 玩家基础属性与数值区间（设计层面）

- 玩家与敌人的数值（HP、伤害、移动速度、射速等）统一在**简单、易于平衡的离散区间**内设计，避免过细微调。
- 玩家基础移动与射击节奏应满足：
  - 初始状态即可在前几波稳定清理小怪。
  - 通过少量升级后，明显感受到输出与生存能力提升。
- 具体数值（如像素速度、射击间隔、伤害点数等）以代码实现为准，本节只约定：
  - **移动速度**：新手可以在合理反应时间内完成“看见 → 躲开”。
  - **射击频率**：一秒内有多次命中反馈，但不形成满屏子弹。
  - **升级成长**：每次升级能带来约 10–30% 的体感提升（而不是极小改动）。

### 玩家主武器与弹种（MVP 三种）

- **基础子弹（主武器 1）**
  - 行为：直线向上飞行，不带追踪与穿透。
  - 使用独立场景：`res://scenes/bullets/PlayerBullet.tscn`。
  - 主要用于前几波清理小怪，数值上偏稳定、泛用。
- **弓箭（副武器，词条解锁）**
  - 首发瞄准最近敌大致方向后直线飞行；命中敌机后消失。
  - **飞行消弹**：轨迹上与 `enemy_bullet` 重叠时仅销毁该敌弹，箭矢不消失。
  - 单发伤害高于主武器子弹（代码层 `damage_bonus` 更高）。
  - 场景：`PlayerArrow.tscn`。
- **回旋镖（副武器，词条解锁）**
  - **解锁 + 回旋镖+1**：三选一「双刃回旋」首次解锁；再次选取齐射数量 +1。场上同时可有多枚（与数量一致）；**全部回收到玩家后**再齐射下一波（无固定冷却）。
  - 命中敌机**不因穿透次数销毁**（与基础子弹不同），仅回程触玩家或出界销毁。
  - 场景：`PlayerBoomerang.tscn`；`Player` 上 `boomerang_speed_mult` / `boomerang_return_speed_mult` 可调。
- **炸弹（副武器，升级解锁）**
  - 行为：按冷却自动向上发射；**弹速低于主武器满速、高于旧版炸弹**（具体倍率以 `Player` 上炸弹 `speed_mult` 为准）；直线飞行；仅在与敌机/Boss 重叠时引爆。
  - **范围伤害（AoE）**；**仅爆炸帧多边形内**清除 `enemy_bullet`（飞行中不再撞消敌弹）。
  - 场景：`res://scenes/bullets/PlayerBomb.tscn` + `PlayerBomb.gd`。
- **符卡弹幕**（与副武器炸弹无关）
  - 发动时自机位置周身高密度径向弹幕；使用 `PlayerSpellBullet.tscn`：**碰到的敌弹单独销毁**，不全场清弹；对敌机/Boss 仍按 `BulletBase` 命中结算。

### 伤害精度与玩家生命（2 条命）

- 玩家拥有**离散生命值系统**：每局固定 **2 条命**（2 Life），不随升级改变。
- 玩家在局内**不会立刻 Game Over**，但当生命归零（Life = 0）时，下一次伤害结算立即结束本局。
- 每次受到伤害（被敌弹命中或撞上敌机/Boss）时，优先结算「稳态护盾」等防护效果；若未被防护，则扣除 **1 条命** 并进入短暂无敌窗口。
- **波次间生命恢复**：每当一波战斗清场并进入升级结算时，若当前剩余生命小于上限 2，则自动恢复 **1 条命**（恢复后不超过 2 条命），在 HUD 中实时更新显示。
- 敌人受击结算支持小数，便于百分比伤害等平衡；UI 展示保留 1 位小数即可。
- 玩家子弹伤害等数值可使用小数（如“伤害 +20%”后 1.0 → 1.2），以保持平衡空间。

### 统一 HP 与伤害结构（EnemyBase / BulletBase）

- **敌人基类 `EnemyBase`**：
  - 属性：`max_hp`（最大生命值）、`hp`（当前生命值）。
  - 方法：`apply_damage(amount)` 处理受到的所有伤害来源（玩家子弹、技能等），在内部扣减 `hp` 并在 `hp <= 0` 时调用 `_on_dead()`。
  - 方法：`_on_dead()` 负责处理敌人死亡后的行为（播放特效、加分、通知波次系统等），由各具体敌人重写。
- **子弹基类 `BulletBase`**：
  - 属性：`damage`（本次命中的伤害数值）、`speed`（子弹移动速度）、`direction`（单位方向向量或角度）。
  - 方法：`set_direction(dir)` 在发射时设定子弹飞行方向。
  - 方法：统一的移动与寿命逻辑（如超出屏幕自动销毁），由基类实现，具体子弹根据需要覆盖部分行为（如弓箭的轻微追踪、回旋镖的折返轨迹）。
- **玩家子弹与敌人交互约定**：
  - 所有玩家子弹（包括基础子弹、弓箭、回旋镖以及后续符卡/炸弹弹幕）都继承自 `BulletBase`。
  - 玩家子弹**不直接修改敌人 HP**，只调用敌人侧的 `apply_damage(damage)` 接口；敌人内部统一处理 HP 扣减与死亡逻辑，便于之后扩展护盾、减伤等机制。

## 受击与判定（休闲容错）

- 受击应有清晰反馈（如短暂无敌/硬直、闪烁、轻震动），但不过分惩罚。
- 玩家每次受击都会明显影响本局表现（例如连击与评分、剩余生命），但不鼓励一击即死。
- **东方式判定点**：玩家受击判定为 **机身中心小圆**（`CircleShape2D`，半径与贴图分离）；**局内始终绘制可见判定点**（内红外白细环，参考东方），与碰撞圆一致，避免“靠猜”。
- **擦弹加分（Graze）**：`GrazeArea` 圆半径约 **132**（`Player.tscn` 局部，可再调）；与 **敌弹 / 敌机（含 Boss）** 重叠时按 **固定间隔**（约 **50ms** / 目标）重复计分；单次分 **`Main.GRAZE_SCORE`**（约 9）；VFX 在 **玩家中心** 实例化 `GrazeSpark`，约 90ms / 目标 节流。**每次擦弹还会为连击条增加少量连击值**（例如等同一发命中所得）；**并且每次擦弹为当前局符卡冷却额外减少 `0.05` 秒（若当前已无剩余冷却则无额外效果）**，鼓励高风险贴弹。

### 玩家受击后的表现变化（含 Combo）

- 玩家受到伤害时：
  - 结算完护盾等防御效果后，若仍有生命则**扣除 1 条命**；
  - 当本次扣命后生命归零（Life = 0）时，本次受击立即触发本局结束并进入结算。
- 受击不再**直接将连击清零**，而是对当前 `Combo` 进行惩罚性衰减：
  - 受击时 `Combo` 降低 **30%**（`Combo := floor(Combo × 0.7)`），向下取整。
  - 若计算结果小于等于 0，则视为连击断档（Combo 清零，触发连击中断反馈）。
- 判定圆小于机体贴图、明显可见（约 **24** 局部半径 × Player `scale`，与 `HitJudgementVisual` 一致）；场景树中 **CollisionShape2D / HitJudgementVisual 排在 Sprite2D 之上** 便于编辑，运行时由 `z_index` 保证判定点仍画在贴图之上。

## 升级（MVP）

- 击杀敌人主要用于统计表现（得分/连击/DPS 等），不驱动升级。
- **升级由波次结束直接触发**：每波清场后默认弹出三选一升级界面，无经验条。

### 推荐节奏参数（整体加快后）

- **刷怪间隔**：普通小怪生成间隔建议控制在约 1.0–1.2 秒（早期波次可略长，后期略短）。
- **波次推进时间**：每一波持续约 30–35 秒后进入下一波；Boss 战总时长仍控制在 45–75 秒区间。

## 敌人基础属性（当前实现目标）

- 小怪（冲锋机）基础属性示例：
  - HP：2
  - 受击：被玩家子弹命中扣 1 点 HP → HP ≤ 0 时销毁
  - 碰撞：与玩家碰撞时触发玩家受击反馈（连击清零等），敌机同时销毁；玩家不扣血、不死亡

## 高连击战斗增益（新增）

- 为了让连击不仅影响结算分数，还能直接影响战斗手感，连击达到阈值后给玩家临时 Buff（受击断连后立即失效）。
- 推荐档位（MVP）：
  - **理论射速** = 连击倍率÷间隔；**实际** 封顶 **75/s**（`Player.max_main_rof`）。溢出 **每 1% → +0.02 伤**；局内 HUD **仅超上限时** 才展示封顶与理论/实际两行，溢出加成写在 **主炮伤害 [+…]**。
  - 连击攻速档位（`set_combo_buff_tier`）：**10 / 25 / 50 / 100 连** 逐级抬高倍率（低档约 ×1.10 / ×1.22 / ×1.34）；**100 连以后**仍每 **100 连** 升一档，**每档固定 +5% 攻速**（倍率 +0.05，线性、可预期）。**50 连 +1 伤**、**100 连 弹速 +15%** 不变。HUD **齐射 a/b 发** 的 **/b** 为齐射上限，与射速无关。
- 设计目标：
  - 普通玩家在 10–30 连区间能明显感到“越打越顺”；
  - 高玩在 50+ 连时能打出更爽快压制，但不应直接破坏 Boss 可读性与躲避需求。
