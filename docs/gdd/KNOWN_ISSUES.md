# 已知问题记录（临时）

- ~~**HP HUD**：已改为右上角数字显示（如「HP: 3」），不再使用心形图标。~~  
  - **更新**：评分制新规则下，局内已完全取消玩家 HP 与生命值系统，玩家不会死亡；仅保留评分相关 HUD（Score/Combo/DPS）。本条所述 HP HUD 方案作废。

- ~~**双发/散射重复升级覆盖**~~（已处理）
  - 处理：改为「弹数+1」单一升级（multi_shot），可叠加，最多 6 发；射出角度按子弹数均匀分布。

- ~~**玩家移动偶发卡顿**~~（已处理）
  - 处理：单帧拖拽位移限制在 120px 内，避免触摸抖动造成大跳；clamp 后把 `_pointer_pos` 同步为当前屏幕坐标，避免与下一帧拖拽冲突。

- ~~**Boss 无法被玩家子弹命中**~~（已处理）
  - 原因：Boss 初始未加入到玩家子弹用于判定受击的敌人分组中（如 `enemy` 组），导致 `player_bullet` 的 `area_entered` 检测不到 Boss，`apply_damage` 不会被调用。
  - 处理：在实现中将 Boss 根节点调整为 `Area2D` 并加入 `enemy` 分组，同时在玩家子弹脚本中兼容 `boss` 分组，使 Boss 能与普通敌人一样被玩家子弹命中并正确掉血。

## 战斗评分/连击/DPS 相关（新设计引入后的暂定问题）

- **评分与连击平衡性未知**（待调优）
  - 问题：目前仅在 GDD 中给出示例连击区间与得分系数，实际体验中可能出现“高玩分差过大 / 普通玩家难以触发高连击”的情况。
  - 验收方向：通过试玩收集数据与主观反馈，使得普通玩家在数局内也能稳定打出 10–30 连左右并获得明显加分，高玩可以挑战 50+ 连与更高评分。

- **DPS 统计频率与 UI 抖动**（风险）
  - 问题：若每帧计算并刷新 DPS，数值可能跳动过快，且在低端设备上有性能风险。
  - 建议方案：采用“最近 N 秒内伤害”的滑动窗口 + 固定刷新间隔（例如每 0.3–0.5 秒更新一次 UI），实现后需重点观察性能与可读性。

- **无 HP 下的紧张感**（需关注）
  - 问题：玩家不会死亡，若仅靠连击与评分惩罚，可能削弱容错与紧张感。
  - 验收方向：通过实际实现与测试，在“玩家不会被踢出游戏”的前提下，仍保持一定压力（例如受击后更难维持高连击与高评分）。
